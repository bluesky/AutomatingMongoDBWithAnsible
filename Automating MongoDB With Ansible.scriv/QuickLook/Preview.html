      • mongodb_status

        mongodb_status


        Ensures the replicaset set is stable in the sense it has a primary and N (usually 2) number of secondaries. The module is useful for ensuring the replicates is stable before proceeding with certain operations. For example we might reboot a MongoDB node after patching the OS. We could use this module to ensure the node comes back up and MongoDB rejoins the replicates. We could then safely proceed with patching the OS on the next node.

      • mongodb_info

        mongodb_info


        This module allows you to extract various system information from a MongoDB instance. For example it is possible to extract instance-level, database-level and user-level information.
